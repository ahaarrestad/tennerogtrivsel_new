# Plan: llms.txt og llms-full.txt

## Mål

Generere `/llms.txt` (kortfattet indeks) og `/llms-full.txt` (komplett innhold) automatisk ved bygg, i tråd med [llms.txt-standarden](https://llmstxt.org/). Samme mønster som `robots.txt.ts`.

## Datakilder

| Kilde | Brukes til |
|-------|-----------|
| `src/content/innstillinger.json` | Klinikkens navn, beskrivelse, adresse, tlf, åpningstider |
| `src/content/tannleger.json` | Tannlegeliste med tittel og beskrivelse |
| `src/content/tjenester/*.md` | Tjenestebeskrivelser (frontmatter + body) |
| `src/content/prisliste.json` | Kategorier og priser |

## Filer som opprettes

```
src/pages/llms.txt.ts         – Astro APIRoute, leverer /llms.txt
src/pages/llms-full.txt.ts    – Astro APIRoute, leverer /llms-full.txt
src/scripts/generate-llms.js  – Genereringslogikk (testbar, som generate-robots.js)
src/scripts/__tests__/generate-llms.test.js
```

## Format

### /llms.txt (kortversjon)
```
# Tenner og Trivsel

> Moderne tannklinikk i Stavanger. <siteDescription>

Adresse: <adresse1>, <adresse2>
Telefon: <phone1>
Åpningstider: <businessHours1–5>

## Tjenester
- <tittel>: <ingress>
...

## Tannleger
- <name> (<title>): <description>
...

## Mer informasjon
- [Tjenester](/tjenester)
- [Prisliste](/prisliste)
- [Kontakt](/kontakt)
- [Fullstendig innhold](/llms-full.txt)
```

### /llms-full.txt (fullversjon)
Alle data fra kortversjonen, pluss:
- Full tjenestetekst (body fra markdown)
- Komplett prisliste gruppert per kategori med priser

## Implementasjonsrekkefølge

1. **`generate-llms.js`** — rene funksjoner som tar parsed JSON/markdown som input og returnerer strenger. To eksporterte funksjoner: `generateLlmsTxt(data)` og `generateLlmsFullTxt(data)`.
2. **`generate-llms.test.js`** — enhetstester for begge funksjoner. Dekker: korrekt seksjonstruktur, håndtering av tomme felt, prisliste-gruppering.
3. **`llms.txt.ts`** og **`llms-full.txt.ts`** — leser filer fra `src/content/`, kaller generate-funksjoner, returnerer `text/plain`-respons.

## Avgrensninger

- Ingen bilder eller HTML — kun markdown/plaintext
- Blokkeres **ikke** for testdomenet (i motsetning til robots.txt) — AI-lesing er nyttig i alle miljøer
- Innhold leses statisk ved bygg (ikke runtime fetch)
- Prisliste vises med format: `<behandling>: <fra-pris> kr` der pris finnes

## Testmål (80 % branch coverage)

- Tom tannlegeliste → seksjonen utelates
- Tjeneste uten body → vises kun med tittel/ingress
- Prisliste med manglende pris-felt → hoppes over
