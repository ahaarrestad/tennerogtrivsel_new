# Gemini CLI Configuration

## Context Loading
At the start of every session, you MUST:
1. Read `CLAUDE.md` to establish project rules, architectural standards, and quality gates.
2. Read `TODO.md` to understand the current project status and active tasks.

All instructions within `CLAUDE.md` are to be treated as foundational mandates for this session.

## Project-Specific Skills
You MUST check `.claude/skills/` for specialized procedures. When performing tasks such as quality gates, commits, coverage analysis, security audits, or updating project status, you MUST read and follow the corresponding `SKILL.md` in that directory to ensure compliance with project-specific workflows.
